# AI Slop Cleaner Skill

Clean up "AI slop" - code that works but has quality issues from rapid AI generation.

## Use When

- "cleanup", "refactor", "deslop"
- Code generated by AI needs polish
- Technical debt from rapid prototyping

## Cleanup Targets

### Code Smells
- [ ] Unused imports/variables
- [ ] Overly verbose code
- [ ] Inconsistent naming
- [ ] Magic numbers/strings
- [ ] Deep nesting (>3 levels)
- [ ] Large functions (>50 lines)

### Anti-patterns
- [ ] God objects
- [ ] Duplicate code
- [ ] Feature envy
- [ ] Shotgun surgery patterns
- [ ] Speculative generality

### Style Issues
- [ ] Inconsistent formatting
- [ ] Missing type annotations
- [ ] Poor error messages
- [ ] Weak documentation

## Process

1. **Identify Scope** - Target files changed in recent AI work
2. **Lock Behavior** - Ensure tests exist before changes
3. **Clean Iteratively** - One smell category at a time
4. **Verify** - Run tests after each pass
5. **Document** - Note what was cleaned

## Usage

```
$ai-slop-cleaner                    # Clean recent changes
$ai-slop-cleaner src/module        # Clean specific directory
$ai-slop-cleaner --review          # Preview only, no changes
```

## Safety

- Never delete tests
- Preserve public APIs
- Keep behavior identical
- Commit between passes
