# โครงสร้างของ Sub-Agent

## 1. หน้าที่หลัก
- ระบุหน้าที่ของแต่ละ sub-agent เช่น การค้นหา, การประมวลผลข้อมูล, การสร้างรายงาน เป็นต้น

## 2. ความรับผิดชอบ
- กำหนดความรับผิดชอบที่ชัดเจน เช่น การเชื่อมต่อกับ API, การจัดการข้อมูล, การตรวจสอบผลลัพธ์

## 3. โมเดลที่ใช้
- ระบุโมเดลที่แต่ละ sub-agent จะใช้ตามความต้องการของงาน เช่น โมเดลสำหรับการค้นหาข้อมูลหรือโมเดลสำหรับการวิเคราะห์ข้อมูล

## 4. ขั้นตอนการทำงาน
- อธิบายขั้นตอนการทำงานของแต่ละ sub-agent โดยจัดแบ่งเป็นงานที่เล็กลงและง่ายขึ้น

## 5. การสื่อสาร
- กำหนดวิธีการสื่อสารระหว่าง sub-agent เช่น การใช้ API หรือการส่งข้อมูลระหว่างกันผ่าน queue เป็นต้น

## 6. การตรวจสอบและประเมินผล
- ระบุวิธีการตรวจสอบผลลัพธ์และประเมินประสิทธิภาพของแต่ละ sub-agent

## 7. ข้อกำหนดทางเทคนิค
- ระบุข้อกำหนดทางเทคนิคที่จำเป็นสำหรับการพัฒนาของแต่ละ sub-agent เช่น ภาษาโปรแกรม, ไลบรารี, หรือเครื่องมือที่ใช้

# ตัวอย่าง

1. **Sub-Agent สำหรับการค้นหา**: ใช้โมเดลค้นหาข้อมูลจาก API ที่กำหนด
2. **Sub-Agent สำหรับการประมวลผลข้อมูล**: ใช้โมเดลวิเคราะห์ข้อมูลและสรุปผล
3. **Sub-Agent สำหรับการสร้างรายงาน**: ใช้โมเดลสร้างเอกสารหรือรายงานจากข้อมูลที่ประมวลผลแล้ว
